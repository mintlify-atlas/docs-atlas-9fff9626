Kohya's GUI can be installed locally using either the `uv` or `pip` method. Both produce the same end result — a running Gradio server with the full GUI — but they differ in how dependencies are resolved and environments are managed. Pick the combination that matches your operating system and tooling preference.

## System requirements

Before installing, confirm your system meets these baseline requirements:

| Requirement | Minimum |
|---|---|
| GPU | NVIDIA GPU (VRAM needs vary by training method) |
| CUDA | NVIDIA CUDA Toolkit 12.8 |
| Python | 3.10.x or 3.11.x (below 3.12) |
| Git | Any recent version |

<Note>
  macOS is only supported through the **pip method**. The `uv` scripts are not tested on macOS and CUDA is generally not required or compatible with Apple Silicon.
</Note>

## uv vs pip — what's the difference?

Both methods install the same packages from `pyproject.toml`. The difference is in the tooling used to create and manage the virtual environment.

| | `uv` | `pip` |
|---|---|---|
| Speed | Significantly faster dependency resolution | Standard speed |
| Isolation | Managed automatically by the launch script | Standard `venv` |
| Upgrades | Re-running the launch script is sufficient | Requires re-running `setup.sh` / `setup.bat` |
| Debugging | Less familiar tooling for some users | Well-documented, IDE-friendly |
| macOS support | Not recommended | Supported |

<Tip>
  If you are unsure which method to use, start with `uv`. If you run into problems, fall back to `pip` — both produce an identical running environment.
</Tip>
