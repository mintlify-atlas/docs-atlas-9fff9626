Kohya's GUI is a user-friendly, Gradio-based graphical interface for [Kohya's Stable Diffusion training scripts](https://github.com/kohya-ss/sd-scripts). It eliminates the need to hand-craft complex command-line invocations — you configure training parameters through a web UI, and the application generates and executes the correct CLI commands for you. Whether you're fine-tuning a base model, building a LoRA adapter, or experimenting with Textual Inversion, Kohya's GUI gives you full control from a browser window.

## Supported training methods

The GUI exposes four core training workflows, each available as its own tab:

- **LoRA (Low-Rank Adaptation)** — Train lightweight adapter networks that modify a base model's behavior without replacing its weights. LoRAs are small, portable, and composable.
- **DreamBooth** — Fine-tune an entire model on a small set of subject images so the model learns to generate that subject on demand.
- **Fine-tuning** — Full or partial weight fine-tuning for broader style or capability adjustments to a base checkpoint.
- **Textual Inversion** — Teach the model a new concept by embedding it in the token space, without modifying any model weights.

## Supported model architectures

| Architecture | Notes |
|---|---|
| SD 1.x | Stable Diffusion 1.4 / 1.5 |
| SD 2.x | Stable Diffusion 2.0 / 2.1 |
| SDXL | Stable Diffusion XL (base + refiner) |
| Flux.1 | Added in v25.0.0 |
| SD3 | Added in v25.0.0 |

## Platform support

Kohya's GUI runs on **Windows**, **Linux**, and **macOS** (macOS via the pip method). Linux support is actively maintained through community contributions; macOS compatibility may vary across hardware generations, particularly Apple Silicon.

## Key features

- Visual interface for the full range of sd-scripts training parameters
- Automatic generation of the underlying CLI commands — inspect them before running
- `config.toml` support to pre-fill default paths across sessions
- Sample image generation during training with configurable prompt files
- Multi-GPU support via the Accelerate launch tab
- Masked loss support for region-targeted training
- Headless / reverse-proxy mode for running on remote servers
